# Test Plan

本文记录 AegisRide Helmet 的测试计划，便于竞赛展示、结题报告和后续迭代。

## 1. 单模块测试

### I2C 扫描

目标：

- 确认 BMI160、MAX30102、AHT20 能被扫描到。
- 确认 I2C 总线没有地址冲突。

记录内容：

- 扫描地址。
- 接线方式。
- 供电方式。
- 失败时的串口日志。

### BMI160

目标：

- 验证加速度和角速度原始值正常变化。
- 验证静止、移动、冲击场景下数据范围。
- 初步标定碰撞阈值。

测试场景：

- 静置桌面。
- 手持轻微晃动。
- 模拟骑行震动。
- 软垫上模拟碰撞。

### MAX30102

目标：

- 验证传感器 PART_ID。
- 验证手指接触和离开时 IR 信号变化。
- 验证 BPM 输出稳定性。

测试场景：

- 未佩戴。
- 手指稳定接触。
- 运动干扰。
- 弱接触或偏移。

### AHT20

目标：

- 读取温度和湿度。
- 验证 10 分钟低频任务不会阻塞主循环。

### GNSS

目标：

- 验证冷启动定位。
- 验证经纬度和速度更新。
- 验证静止速度滤波。

测试场景：

- 室外空旷环境。
- 校园道路移动。
- 弱信号区域。

### MQTT

目标：

- 验证 SIM 卡、网络注册、Broker 连接。
- 验证每秒发布 payload。
- 验证断网重连。

记录内容：

- 联网耗时。
- 首次连接耗时。
- 发布间隔。
- 重连日志。

## 2. 集成测试

| 测试项 | 预期结果 |
| --- | --- |
| 正常启动 | 初始化 I2C、BMI160、MQTT、Timer2、心率、GNSS、温湿度 |
| 正常骑行 | 数据周期上传，小程序刷新正常 |
| 碰撞模拟 | 蜂鸣器报警，MQTT `collision=1`，小程序弹窗 |
| 按键解除 | 本地蜂鸣器停止，碰撞锁存清除 |
| 省电模式 | 心率/GNSS/温湿度暂停，碰撞检测和 MQTT 保留 |
| 恢复正常模式 | 心率/GNSS/温湿度恢复采集 |
| 网络断开 | MQTT 线程重连，不阻塞主循环 |

## 3. 外场测试

建议至少完成以下场景：

- 静态室内桌面测试 30 分钟。
- 校园道路慢速骑行 10 分钟。
- 颠簸路面骑行 10 分钟。
- 模拟碰撞 5 次。
- 省电模式运行 30 分钟。
- 弱信号区域数据上传测试。

## 5. 验收标准

- 系统启动后主循环持续运行，无明显阻塞。
- 碰撞检测在模拟冲击后能触发告警。
- 非碰撞普通震动不应频繁误报。
- MQTT 数据能被云端稳定接收。
- 小程序能显示最新状态和告警。
- 用户可通过按键或小程序解除告警。
